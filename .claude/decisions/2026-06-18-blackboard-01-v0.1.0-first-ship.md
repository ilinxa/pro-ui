# blackboard-01 v0.1.0 — first ship

The **55th procomp**: a dark-navy collaborative **chalkboard** where a team writes handwritten notes. Greenfield, single-session GATE 1 → 2 → 3. Category `data` (revalidated — the collaborative-feed family lives there).

## What it is

A vertical stream of handwritten notes, each in the author's chosen **ink color + chalk width + handwriting font**. Auto-save (debounced, no Save button) with optimistic post → reconcile → retry; **scroll-up lazy load** (10/page, scroll-anchored jump-free prepend); inline **author-on-hover** label; **pin** to a sticky top row; **`@mentions`** over a team roster (+ a `@you` emphasis on notes mentioning the viewer); a handwritten **chalk-red unread** marker; and a **themeable** board surface (solid color or custom image with a legibility overlay).

## Key decisions

- **Compound (per the rule).** Headless `BlackboardRoot` (owns all state + context + imperative handle) + 7 flat context parts (`Surface`/`PinnedRow`/`NoteStream`/`NoteItem`/`Composer`/`NotificationBadge`/`BackgroundEditor`) + 9 context-free Tier-C primitives (`HandwrittenNote`/`NoteComposer`/3 pickers/`MentionPicker`/`MentionText`/`UnreadCount`/`BoardBackground`) + a logic-free `Blackboard01` assembly. **Drop the composer ⇒ a free read-only board** (demo's "Lighter" tab proves the subset). Flat exports only.
- **Composes NO other procomps** — only `button` + `textarea` shadcn primitives. This is the first ship since the composition cluster (content-composer / media-* / media-library) with **zero** cross-procomp rewriter risk (no `@/registry/.../types` to mangle, no `registryDependency` chain).
- **Portability forces the API.** No network I/O: auto-save = `onDraftChange`/`onPostNote` (return `Promise<BlackboardNote>` for reconcile); lazy = `onLoadOlder(beforeId, 10)`; real-time = controlled `notes` + imperative `appendNote()`; mentions/notifications = `onMention` data only.
- **Pin source-of-truth** (GATE-2 deep-review fix): `pinnedNoteIds` (controlled) wins; else `note.pinned` (uncontrolled). Pinned notes lift out of the stream into the sticky row.
- **Hybrid fonts (Q4).** 4 bundled `@fontsource` faces — **Kalam** (real 300/400/700) + Caveat + Patrick Hand + Shadows Into Light — self-loaded by `BlackboardRoot` (side-effect import) and exposed via `--bb-font-*` CSS vars with a `cursive` fallback; overridable via the `fonts` prop. "Chalk width" = real font-weight where the face has weights, faux `-webkit-text-stroke` otherwise.

## Design-mandate exceptions (3, scoped + GATE-3 design-dim verified)

The strict mandate (Onest/JetBrains, signal-lime accent, no off-palette color) gets three user-requested, contained exceptions: (1) handwriting font on **note text + the unread number only** (all chrome stays Onest); (2) the board is **dark-navy in both light and dark themes** (chalkboard identity; own `--bb-*` surface tokens, doesn't inherit `--background`); (3) a **chalk-red unread number** `oklch(0.66 0.19 22)` (chroma 0.19 ≤ 0.20). Signal-lime remains the interactive accent. Ink palette tones all chroma-capped.

## Post-build refinement (same session, user feedback)

After the first walkthrough the user asked for a **more minimalistic composer** that **appears on double-click** rather than being always-docked. Folded into v0.1.0 (still pre-push): new `composerMode?: "always" | "double-click"` (**default `"double-click"`**) on the context + props; `BlackboardSurface` opens the composer on double-click (with a faint discoverability hint), `BlackboardComposer` renders only while open and auto-focuses on reveal with Esc / ✕ to dismiss; the composer itself was decluttered (no tray fill, borderless underline textarea, dividers removed, pickers tightened). `composerMode="always"` restores the docked bar. Re-verified: tsc 0 / lint 0 / registry:build 0 / dev SSR 200 (composer correctly absent from the default tab; hint present).

## v0.1 scope calls

- **Live in-textarea mention highlight → v0.2.** v0.1 inserts `@name` as plain text and styles mention tokens on *rendered* notes (`MentionText`); the `@`-picker + storage + `onMention` + `@you` all ship. The highlighted-textarea mirror overlay (cosmetic, riskiest piece) is deferred.
- **Pickers are plain `Button`/`button` groups** (no `toggle-group`) and the background editor is an **inline panel** (no Popover) — deliberately shrinking the F-cross-13 (Radix↔Base-UI) surface to just `button` + `textarea`. Mention popover is anchored to the composer, not `PopoverAnchor` (Base UI lacks it).

## Lessons (reusable)

1. **meta-deps import-regex quirk:** the validator's `import…from` pattern is lazy and spans from the first `import` to the first ` from `, so side-effect imports (`import "@fontsource/x.css"`) that *precede* a `from` import in the same file get swallowed → reported as phantom-npm. **Fix: put the `from` import FIRST** so the side-effect imports have no trailing `from` to be consumed by. (Generalizes the prior "meta-deps regex false-positive" memory.)
2. **`react-hooks/refs` on bundled context:** reading refs (or calling functions / rendering JSX that captures refs) **off a context value during render** trips the rule, because the value object also carries `scrollRef`/`sentinelRef`. **Fix: destructure the context into locals at the top** of the part and attach refs only in `ref=` **prop form** (re-confirms the media-editor v0.1.2 lesson).
3. **Optimistic-vs-controlled reconcile without a prune effect:** deriving display via a `useMemo` dedup (controlled `notes` wins over local extras) keeps render correct and avoids `react-hooks/set-state-in-effect`; cap the local buffer instead of pruning in an effect.
4. **Relative-time + locale `title` need `suppressHydrationWarning`** (rich-card v0.4.3 class) — applied on the author label.
