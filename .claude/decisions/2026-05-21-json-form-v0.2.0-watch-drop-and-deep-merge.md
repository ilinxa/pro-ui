The behavioral counterpart to the additive v0.1.7 substrate work — lifts the per-keystroke render ceiling and fixes a long-standing `defaultValues` shallow-replace footgun. Strictly auto-applied; zero consumer code changes required.

## Why this matters

v0.1.7 set up the substrate (`BUILTIN_RENDERER_TYPES_SKIPPING_ALL_VALUES` whitelist defined-but-unused, `dependsOn` typed prop with no runtime effect). v0.2.0 turns it on.

Expected perf delta (20-field form, 1 keystroke):
- FieldWrapper renders: **20 → 1–4** (only fields whose narrow-deps `useConditional` / `useComputed` reference the typed field)
- Default-registry `useWatch` subscriptions per field: **1 (full-bag) → 0 (snapshot)**

The deep-merge fix is the kind of latent bug that's invisible until a consumer hits it: a v0.1.x form-level `{ address: { city: 'NYC' } }` silently dropped any per-field `defaultValue` at `address.country`. The fix is what consumers expected the behavior to be.
