## F-02 — cross-backend consumer-tsc smoke: 3 real Base UI errors → FIXED

Smoke = install `@ilinxa/calendar-01` from the live registry into the `base-nova` (Base UI) consumer at `e:/tmp/ilinxa-smoke-consumer`, then `pnpm tsc --noEmit`. Install succeeded; consumer-tsc failed with **3 errors** — all the Radix↔Base UI divergence the smoke exists to catch (and all latent since **v0.1.0**, whose smoke was also deferred):

1. **`calendar-toolbar.tsx` — `ToggleGroup value`** — Base UI's `ToggleGroup` is a *multi-value* model (`value: string[]`), so passing a single `CalendarView` string fails (`TS2322`). Radix's `type="single"` takes a string.
2. **`calendar-toolbar.tsx` — `ToggleGroup onValueChange`** — Base UI's signature is `(groupValue: string[], eventDetails) => void`, not `(v: string) => void`. (Same family as the todo-rich-card R10 `Select` lesson.)
3. **`calendar-root.tsx` — `TooltipProvider delayDuration`** — `delayDuration` is a Radix-only prop; Base UI's `TooltipProvider` takes `delay`, so the prop doesn't exist there (`TS2322`).

### Fixes

- **View switcher → plain-button segmented control.** Replaced `ToggleGroup`/`ToggleGroupItem` with a `<div role="group">` of `<button aria-pressed>` elements (the proven gantt-timeline-01 zoom-switcher pattern; the blackboard "pickers are plain Buttons to shrink the F-cross-13 surface" rule). `setView(v)` takes `CalendarView` directly — no cast. This **drops the `toggle-group` dependency entirely** (meta + registry regdeps), shrinking the cross-backend surface by one primitive.
- **Dropped `delayDuration={250}`** from `<TooltipProvider>`. The default delay is cross-backend-safe; there's no common prop name for a custom delay (Radix `delayDuration` vs Base UI `delay`). Minor UX delta only.

### Validation

- **Pre-deploy:** patched the two fixed files into the existing Base UI consumer install and re-ran `pnpm tsc --noEmit` → **exit 0, 0 errors** (was 3).
- **Post-deploy (canonical, 4-ship pattern): ✅ CLEAN.** After Vercel deployed v0.2.1, a fresh `pnpm dlx shadcn@4.6.0 add @ilinxa/calendar-01` from the **live registry** into the `base-nova` (Base UI) consumer installed clean (`INSTALL_EXIT=0`) and `pnpm tsc --noEmit` returned **0 errors** — also exercising the shadcn rewriter on the deployed artifact. The full loop is closed: ship v0.2.0 → smoke (3 errors) → patch v0.2.1 → re-smoke live (0 errors).
