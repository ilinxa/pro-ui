# calendar-01 v0.1.0 — first ship (the date-grid sibling; completes the task set)

The **57th procomp** and the **fifth member of the task-management family** — the date-grid
counterpart to `gantt-timeline-01`'s continuous axis, consuming the **same canonical
`TodoItem[]`** as todo-rich-card / todo-tree / kanban-board-01 / gantt. User named it to
"complete the task set." Greenfield; the gantt description had already reserved it as the
"planned sibling `calendar-01` (next procomp)."

## What shipped (v0.1.0, read-only)

Four views: **Month** (multi-day spanning bars + chips + "+N more" overflow), **Week** + **Day**
(hour time-grids — all-day band + lane-packed timed blocks + now-line), **Agenda** (day-grouped
list). Period nav + view switch + optional jump-to-date mini-nav (shadcn `calendar`), native-title
hover tooltips (`renderTooltip` lazy-embeds the full `<TodoRichCard>`), selection, keyboard
shortcuts (M/W/D/A · ←/→ + PageUp/Down · T). Status/urgency color **imported from todo-rich-card**
(`RAMPS`). SSR-safe `now`. The v0.2 editing surface (editable / permissions / onChange / CRUD
events) is **declared but inert** so v2 is additive.

shadcn-style **compound**: `Calendar01Root` (headless provider) + flat view parts (Toolbar /
MonthView / WeekView / DayView / AgendaView / MiniNav) + Tier-C primitives (EventChip / EventBar /
TimeBlock / MonthDayCell / TimeGrid / TimeGutter / NowIndicator / AgendaRow / EventTooltip /
Skeleton / FullCardTooltip) + the `Calendar01` assembly. Each view is its own module → a
month-only hand-assembly drops the week/day time-grid code; `CalendarFullCardTooltip` is
`React.lazy` so the default tooltip never bundles todo-rich-card.

## Key decisions

- **D5 — all-day/timed classification (the headline GATE-2 question), three-layer first-match:**
  (1) consumer `classifyEvent(item)` predicate; (2) **date-only strings** (`YYYY-MM-DD`, no `T`)
  ⇒ all-day, parsed as a **floating local date** (`new Date(y,m-1,d)`, NOT `Date.parse` — which is
  UTC-midnight and renders a day early in negative offsets); (3) span heuristic (no end ⇒ milestone,
  unless a date-only start ⇒ all-day; span ≥ 1 day ⇒ all-day; else timed). **No fork of the shared
  `TodoItem`.** "Do it like Google" = the date-vs-dateTime mechanism, adapted onto the existing ISO
  string fields. All-day end is **exclusive at local midnight** (iCal/Google).
- **Cross-procomp reuse mirrors gantt:** import `TodoItem` + options/permissions/events from
  `../todo-rich-card`, re-export from `types.ts`; `RAMPS` from `../../todo-rich-card` (FIRST
  `from`-import in color.ts so meta-deps registers it). Calendar additionally re-exports
  `TodoPermissionReason` (gantt's barrel omits it; used by `onPermissionDenied`).
- **Six pure lib modules** (Vitest-ready): `classify` · `color` · `date-range` · `occurrences` ·
  `segments` (month lane layout) · `lane-pack` (week/day overlap columns). Finite-date guards
  throughout (gantt v0.3.1 G2 lesson — unparseable date ⇒ `invalid`, label-only, never throws).
- **Default tooltip = native `title`** (cheap, accessible, zero divergence); shadcn `Tooltip` only
  on the opt-in `renderTooltip` path (TooltipProvider mounted in the Root).
- **SSR-safe now** via seed + **rAF** post-mount (not synchronous setState-in-effect — avoids the
  `react-hooks/set-state-in-effect` rule the blackboard lesson flagged).
