# json-form v0.2.1 — bundled JSDoc + devtools patch

## Summary

Patch bump closing 4 of the 5 open follow-ups from the v0.2.0 GATE 3 full-checklist review in a single commit. No public-API additions, no behavioral changes for any existing v0.1.x / v0.2.0 consumer. One small internal-only export (`useJsonFormContextOptional`) added in `json-form-context.tsx` — used only by the devtools body; NOT exported from the package barrel.

| Finding | File | Change |
|---|---|---|
| F-01 — `dependsOn` JSDoc | `types.ts` | Rewrites the `FieldDefinition.dependsOn` JSDoc — drops the v0.1.7-only future-framing, documents all three semantic states (undefined / [] / non-empty), notes the FINAL renderer identity check, adds explicit stable-identity guidance ("module-scope const or `useMemo`-cache"). |
| F-02 — `allValues` snapshot freshness | `types.ts` | Adds JSDoc to `FieldRendererArgs.allValues` covering snapshot / narrow / fullbag modes. Snapshot mode notes the freshness contract (fresh on own value change via Controller, stale across other fields) + recommends `useJsonFormFieldValue` / `useJsonFormFieldsValue` for live values in snapshot-mode renderers. |
| F-06 — devtools soft-warning | `json-form-context.tsx`, `parts/json-form-devtools-body.tsx` | New internal `useJsonFormContextOptional()` accessor returns `null` instead of throwing. Devtools body short-circuits to a "No JsonForm context found" panel with a recover-action explanation instead of crashing the component tree when consumer forgets the `<JsonFormProvider>` wrap. Hooks-order rule preserved (the optional accessor is called first; tabs only mount when ctx is non-null). Bumped the devtools footer version label `v0.1.7` → `v0.2.1` while there. |
| F-08 — BigInt replacer | `parts/json-form-devtools-body.tsx` | `prettyReplacer` returns `${value.toString()}n` for `typeof === "bigint"` values; `JSON.stringify` itself throws on bigint so any form with a bigint in state would crash the devtools panel before v0.2.1. |
