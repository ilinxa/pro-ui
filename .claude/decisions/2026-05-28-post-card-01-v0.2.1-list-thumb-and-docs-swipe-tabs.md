## 1. post-card-01 v0.2.0 → v0.2.1 — list-variant image width shrink

**Problem:** On narrow containers (column / sidebar / true mobile), the list-variant image column at `w-20 sm:w-28 md:w-32 lg:w-40` dominated horizontal space — at viewport widths where the docs-site rendered the card ~280px wide, the image took ~28% on mobile and grew to ~50% at `md:`, crushing the text column. The post-header's name span collapsed to "H…" because avatar + gap + kebab consumed most of the residual width.

**Fix:** Shrink the image column at every breakpoint while preserving the responsive step:

| | Before | After |
|---|---|---|
| mobile | `w-20` (80px) | `w-16` (64px) |
| `sm:` | `w-28` (112px) | `w-20` (80px) |
| `md:` | `w-32` (128px) | `w-24` (96px) |
| `lg:` | `w-40` (160px) | `w-28` (112px) |

Single line of CSS in [parts/list-variant.tsx:112](../../src/registry/components/data/post-card-01/parts/list-variant.tsx#L112).

**Why patch, not minor:** Visual-only refinement. No public-API change. No new behavior. Meta description string "v0.2.0 — Responsive sweep: padding / avatar / body-text / list-thumb step at sm / md / lg" still accurate (just at smaller absolute values). Per [readiness-review rule](../rules/readiness-review.md), patch bumps (`v0.x.y → v0.x.y+1`) do NOT trigger GATE 3.

## 2. Docs-site `<SwipeTabsList>` wrapper + 39-demo sweep

**Problem:** Many procomp demos declare 5–13 tabs in their `<Tabs>` skeleton. shadcn's `TabsTrigger` has `flex-1` baked in, so children compress to share container width while `whitespace-nowrap` text overlaps visually — most visible on `post-card-01` (13 tabs after v0.2.0). User-reported on mobile preview: overlapping unreadable tabs. Worse: each demo had ad-hoc `flex flex-wrap` / `grid grid-cols-N` overrides that solved overlap by wrapping to 2 rows (also unwanted) or just hid the problem on wide screens.

**Solution:** New docs-site-only wrapper at [`src/components/site/swipe-tabs-list.tsx`](../../src/components/site/swipe-tabs-list.tsx). Single source of truth for the docs-preview tab strip layout. Adopts the proven `useDragScroll` pattern from [`engagement-bar-01/parts/likers-strip.tsx`](../../src/registry/components/data/engagement-bar-01/parts/likers-strip.tsx) (the same procomp the user authored 2026-05-27).

**Behavior:**
- `inline-flex w-max` `<TabsList>` — tabs at natural content width (override `flex-1` with `flex-none px-3`). Stable widths → predictable snap targets.
- Outer scroll container with `overflow-x-auto` + `snap-x snap-mandatory` for native (touch / trackpad) interactions.
- JS mouse-drag pattern from LikersStrip — pointer-capture, cursor-grab→grabbing, drag updates `scrollLeft` live, `onClickCapture` swallows the click when gesture moved > 3px so tab activation only fires on real clicks.
- JS-driven snap-to-nearest on `pointerup` and 120ms after wheel idle — uses `getBoundingClientRect` (viewport-relative, robust to positioning context; the previous attempt with `tab.offsetLeft` was broken because the offsetParent chain didn't include the scroll container).
- Wheel → horizontal scroll for desktop users without trackpad.
- Scrollbar visually hidden (`scrollbar-width: none` + WebKit override + legacy Edge).

**Iteration history (kept for future reference — multiple failed attempts before landing this):**
1. Inline className blob with `flex w-full overflow-x-auto sm:grid sm:grid-cols-[repeat(auto-fit,minmax(96px,1fr))]` — failed because `TabsTrigger.flex-1` made each child take 1/N of container, no overflow engaged.
2. Extract wrapper with same auto-fit grid — fixed mobile but at desktop wrapped to 2 rows.
3. `grid-flow-col auto-cols-[minmax(80px,1fr)]` — fixed wrap but introduced unreachable last snap point + JS drag math errored due to dynamic widths.
4. Fade-mask + CSS snap only — hid mid-tab cuts visually but no PC swipe.
5. Pointer drag + `tab.offsetLeft`-based snap — math bug (offsetLeft was body-relative, not container-relative).
6. Removed drag, kept CSS snap + wheel handler — first-tab still cut on small screens.
7. **Final (this commit) — `inline-flex w-max + flex-none px-3` + drag + `getBoundingClientRect`-based snap.** Stable tab positions, drag works on PC, touch swipes natively with CSS snap, scrollbar hidden, mouse-wheel translates to horizontal scroll.

**Sweep scope:** 39 `demo.tsx` files migrated `TabsList` → `SwipeTabsList`. Ad-hoc `flex flex-wrap` / `grid grid-cols-N` classNames stripped (wrapper handles responsive layout uniformly). Mechanical edit; one sub-agent run + one missed file (`properties-form/demo.tsx`) caught + fixed manually.
