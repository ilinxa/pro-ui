## Roadmap

Next candidates, ordered by team utility:

1. `data/stat-card` — value + label + delta + sparkline. Universal in dashboards.
2. `feedback/empty-state` — icon + title + body + primary action.
3. `forms/multi-select` — combobox with tag chips (shadcn has Command, no real multi-select).
4. `layout/page-header` — title + breadcrumbs + actions slot.
5. `feedback/notification-feed` — grouped, time-bucketed, read/unread.
6. `navigation/command-palette` — cmd+k, grouped results.
7. `media/dropzone` — drag-drop + progress + previews.

**`rich-card` v0.2 follow-ups** (deferred from v0.1.0, by phase per [rich-card-procomp-plan.md](../docs/procomps/rich-card-procomp/rich-card-procomp-plan.md)): v0.2 inline editor (add/edit/remove fields/cards/predefined-key elements, granular change events, dirty tracking, explicit selection); v0.3 drag-drop (2 scopes), bulk ops, per-level/per-card/per-key permissions, custom predefined-key registration, virtualization if needed; v0.4 sync validation hooks + per-commit undo/redo; v0.5 (deferred indefinitely, separate companion) markdown ↔ JSON adapter. The v0.1 API surface is designed so each phase adds props rather than changing them.

**`workspace` v0.2 follow-ups** (deferred from v0.1.0): detach-to-OS-window for multi-monitor users; touch / pen gesture support; aligned-edge linked resize (multiple edges move together); DnD between areas; cross-breakpoint transition animation. The v0.1 API surface is designed so v0.2 doesn't break it.
