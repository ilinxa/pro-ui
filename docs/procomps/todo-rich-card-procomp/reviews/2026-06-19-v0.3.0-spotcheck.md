### Rotating dimension

- **Chosen:** Public API (with the controlled-mode normalize gotcha verified under it).
- **Why:** the headline is three additive public surfaces (`statusEditable`, `TodoStatusOption.tone`/`icon`, `TodoItem.priority`/`labels` + `priorityOptions`/`labelOptions`) plus a near-internal signature change (`StatusBadge {status}`→`{node}`). Backward-compat + cross-backend + the closed-schema gotcha are the risk surface.
- **Verified?** yes
- **Notes:**
  - **Backward-compat:** all new props/fields are optional; `statusEditable` defaults `true` but only *renders* the dropdown when `editable` + `perms.edit` + non-empty `statusOptions`, so non-editable consumers are unchanged. `tone`/`icon` absent → no overlay, time border as before. `priority`/`labels` absent → `CardMeta` renders nothing. The only signature change (`StatusBadge`) is an internal part, not a public export.
  - **B7-gotcha (load-bearing):** `priority`/`labels` added to `normalizeNode`'s authoritative allow-list AND `denormalize`'s `optionalKeys`; a one-line "allow-list is authoritative" comment now flags the step for future field additions. Verified the controlled-mode reconcile round-trips the new fields (they survive `normalize`→`denormalize` so `sync-tree` no longer strips them).
  - **B4 collapse:** auto-collapse is a *derived* render value (`collapsed = isTerminal || ctx.isCollapsed(id)`), NOT a dispatched state change — avoids the toggle-only `toggle-collapse` double-toggle trap; no reducer action added.
  - **Dispatch parity:** the dropdown reuses the exact `edit-field` dispatch + `fieldEdited`/`statusChanged` events as `edit-inline.tsx`'s `commit()`.

## Findings

#### F-01 — Post-deploy cross-backend consumer-tsc smoke (4-ship pattern)

- **Severity:** 🔸 Medium
- **Dimension:** Registry distribution / cross-backend portability
- **Location:** `parts/status-badge.tsx` (`DropdownMenuTrigger asChild`)
- **Observed:** New `asChild` dropdown trigger on the status badge. Producer-tsc/build pass, but Base UI consumers are not exercised by them. Same risk class as the engagement-bar Popover sub-trap and the R10 Select narrowing.
- **Suggested fix:** Run F-cross-11 path-b smoke from the base-nova/Base UI consumer after deploy; patch to v0.3.1 if it surfaces a divergence. `action-menu.tsx` ships the same `asChild` pattern and passed prior smoke, so the expectation is clean.

#### F-02 — B6 was a MISMATCH against current source (report wrong vs our code)

- **Severity:** 🔹 Low → **Closed (no action)**
- **Dimension:** Public API / theming
- **Location:** `parts/card.tsx`
- **Observed:** The note claimed the card *surface* is painted via inline `style.backgroundColor`, blocking consumer `bg-*` / class-based dark mode. Current source already uses the `bg-card` className token; only the *border color* is inline (by time-engine design). No change needed. Mirrors the R10 lesson: a report finding can be flat wrong against our backend.
- **Suggested fix:** None. Documented in the guide that the time engine intentionally owns the inline border color.

#### F-03 — Terminal overlay a11y

- **Severity:** 🔹 Low
- **Dimension:** Accessibility
- **Location:** `parts/card.tsx` (done/blocked overlay)
- **Observed:** The overlay title `<p>` + icon `<span>` are `aria-hidden` (decorative duplicates); the name is announced via the article `aria-label` + header `h3`, and the status via the overlay `StatusBadge` `role="status"`. Overlay sits at `z-20` over the header, so when terminal the header chevron is non-interactive — intended (status changes via the overlay badge).
- **Suggested fix:** None for v0.3; revisit if a consumer needs a keyboard path to expand a terminal card without changing status.

#### F-04 — Prior v0.2.0 follow-ups (person/image/link editors; SSR deterministic paint) NOT in this cohort

- **Severity:** 🔹 Low (scope note)
- **Dimension:** Public API / Robustness
- **Observed:** The v0.2.0 review targeted F-01 (person/image/link editors → re-add to `TodoEditableField`) and F-02 (SSR `useSyncExternalStore` first-paint) at "v0.3.0". This v0.3.0 is scoped to the consumer **integration note** (B-series), not those — they are explicitly carried forward, not silently dropped.
- **Suggested fix:** Re-target both to a later minor (v0.4.0).
