## Findings

#### F-01 — Smoke harness run deferred to post-push

- **Severity:** 🔹 Low
- **Dimension:** Verification
- **Location:** `e:/tmp/ilinxa-smoke-consumer/scripts/smoke-all.mjs`
- **Observed:** Pre-push smoke pulls v0.1.1 from Vercel (current live), not v0.2.0 source. Real smoke verification requires push + Vercel redeploy first.
- **Suggested fix:** Run `node scripts/smoke-all.mjs --slug post-card-01` within ~10 min of pushing C12 to `master`; record the result in `results/2026-05-27-post-card-v0.2.0-smoke.md`. If install or tsc fails, hold off the v0.2.0 announcement until a patch commit (v0.2.1) closes the failure.
- **Owner:** Author (post-push step, same session).
- **Bump target:** No version bump implied — smoke is verification, not source.

#### F-02 — Sub-export asymmetry across v0.2.0 parts

- **Severity:** 🔹 Low
- **Dimension:** Public API
- **Location:** `src/registry/components/data/post-card-01/index.ts`
- **Observed:** Of the 7 new v0.2.0 parts, 4 are sub-exported (`MentionText`, `TagChips`, `RepostOfCard`, `PollWidget`) and 3 are not (`SensitiveGate`, `LinkPreviewCard`, `EditedSuffix`/`PinnedBadge`/`VisibilityBadge` — the last three are decorative-only and consumers can re-derive). The decision was per-part during authoring (sub-export when a host has a plausible standalone use case) but the resulting surface isn't symmetric.
- **Suggested fix:** v0.2.x decision pass — either promote `SensitiveGate` + `LinkPreviewCard` to sub-exports (matches the MentionText/TagChips pattern) OR remove `RepostOfCard` + `PollWidget` sub-exports for consistency. Currently leaning toward promotion (default to open API surfaces per dynamicity-primacy memory).
- **Owner:** Author.
- **Bump target:** v0.2.1 (patch — additive sub-exports don't change consumer behavior).

#### F-03 — Multi-select polls fall through to single-vote UI

- **Severity:** 🔸 Medium
- **Dimension:** Public API
- **Location:** `src/registry/components/data/post-card-01/parts/poll-widget.tsx`
- **Observed:** `PostPoll.multiSelect: boolean` is in the type but `PollWidget` ignores it — when `true`, the widget still renders single-vote UI. Documented in the widget's JSDoc + the guide, but the type surface accepts `multiSelect: true` without warning.
- **Suggested fix:** Either (a) implement multi-select UI (vote buttons stay visible after first vote; viewer can flip multiple options; mirror becomes `optionIds: string[]`); or (b) drop `multiSelect` from `PostPoll` until implementation lands. Authorial preference: (a) in v0.3 — clean implementation including the local-mirror shape change.
- **Owner:** Author.
- **Bump target:** v0.3 (multi-select implementation is a breaking type change for the local mirror).

#### F-04 — `closesAt` countdown doesn't auto-refresh

- **Severity:** 🔹 Low
- **Dimension:** Public API
- **Location:** `src/registry/components/data/post-card-01/parts/poll-widget.tsx`
- **Observed:** `closesAtLine` is computed at render time using `new Date()` for "now". The string is correct on each render but doesn't tick on a timer — the displayed "closes in 3 hours" stays static until the parent re-renders for another reason. Matches the existing `formattedTime` convention in `post-card-01.tsx:168` (same project-wide pattern).
- **Suggested fix:** No-op for v0.2.0 — convention matches existing relative-time displays. Future v0.3 could add a `useRelativeTimeTick(interval = 60_000)` hook used across post-card-01 + comment-thread-01 + likers-strip for a unified ticking mechanism. Cross-procomp concern.
- **Owner:** Library author (cross-procomp).
- **Bump target:** v0.3+ (cross-cutting; not just post-card-01).

#### F-05 — Hardcoded English aria-label in RepostOfCard

- **Severity:** 🔹 Low
- **Dimension:** Public API (i18n)
- **Location:** `src/registry/components/data/post-card-01/parts/repost-of-card.tsx:71`
- **Observed:** `aria-label={`Open ${originalPost.author.name}'s post`}` is hardcoded English. Other i18n surfaces (kebab labels, poll labels, sensitive heading) flow through `DEFAULT_POST_CARD_LABELS` + the consumer's `labels` prop.
- **Suggested fix:** Add `openRepostLabelTemplate?: string` to `PostCard01Labels` with default `"Open {{name}}'s post"`; thread through variant-shared → RepostOfCard. Pattern matches the existing `pollClosesAtLabel.replace("{time}", …)` style.
- **Owner:** Author.
- **Bump target:** v0.2.1 (patch — additive label key).
