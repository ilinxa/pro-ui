## Rotating dimension — Robustness

- **Backward-compat (the hard constraint):** `editable` defaults `false`; every edit branch in the body falls through to the v1 pan path when `!editable` or permission denies; gutter DnD draggables self-disable; the edit popover only mounts when `editable`. The build's non-editable demo tabs render identically. ✅ (F-01 smoke confirms cross-backend.)
- **Gesture arbitration:** `pointerdown` hit-test order is explicit (resize edge → bar body → empty row → pan); right-click (`button === 2`) is excluded from the pointer gesture so it can't start a pan/drag; snap unit resolved once at drag-start (no mid-zoom jump). ✅
- **Mutation correctness:** pure `edit-mutations` use structural sharing + return the same ref on no-op; `setWindow` enforces `end ≥ start + MIN`; `removeItem` returns the removed subtree + parent; circular drops banned via `isAncestor`. Controlled-echo (`onChange(forest)`) means no internal data state to desync. ✅
- **Permissions:** `evalGanttPermission` mirrors the proven todo-tree resolver; `can()` is the single gate for affordances AND every dispatcher veto-before-dispatch; `locked` blocks all; summary bars carry no `data-itemid` so they're non-manipulable. ✅
- **Residual robustness gaps:** sibling-reorder index off-by-one in one direction (F-02); virtualization not suspended during a gutter drag (F-03); draw-create is permission-gated at commit, not start (F-04). All Low.

## Findings

### F-01 — Post-deploy cross-backend consumer-tsc smoke (⚠️ High-ish / Med) — REQUIRED
- **Location:** distribution. **Two new shadcn primitives** (`context-menu`, `input`) → F-cross-13 (Radix↔Base UI divergence) re-arms; `context-menu`'s `asChild` trigger is the specific watch item. Can't run pre-deploy.
- **Fix:** after push + Vercel deploy, run `shadcn add @ilinxa/gantt-timeline-01` into the base-nova/Base UI consumer + `tsc --noEmit`; patch v0.2.1 per the 4-ship pattern if it bites. `@dnd-kit/core` is already a consumer peer (todo-tree/kanban), so only the two new primitives are novel.

### F-02 — Sibling-reorder off-by-one (🔹 Low) — ✅ FIXED pre-ship
- `moveItem` removes-then-inserts at `newIndex`; moving a row DOWN within the same parent shifted the target by one. **Fixed** in `moveItemAction`: decrement when same parent and `fromIndex < newIndex` (and the `onItemMoved.newIndex` reports the adjusted value). Reparent + cross-parent were already correct.

### F-03 — Virtualization not suspended during gutter drag (🔹 Low → v0.2.1)
- If a dragged row scrolls out of the virtual window mid-drag, @dnd-kit loses the active node. Bounded lists are unaffected; deferred (todo-tree suspends — mirror it via a `suspended` flag on `use-gantt-virtual`).

### F-04 — Draw-create gated at commit, not start (🔹 Low → v0.2.1)
- A draw on an empty row always starts; `createItem` enforces the parent's `addChildren` rule at commit, so a denied create draws-then-nothing. Acceptable; tighten by gating the draw-start on the parent's create permission.

### Carried from v0.1.0 (still open)
- **F-02 (v0.1.0)** chart `height` prop (Low, v0.2.x). **F-04 (v0.1.0)** context-split perf (Low, deferred). **F-03 (v0.1.0)** Vitest (the new pure `edit-mutations` + `edit-permissions` are test-ready).
