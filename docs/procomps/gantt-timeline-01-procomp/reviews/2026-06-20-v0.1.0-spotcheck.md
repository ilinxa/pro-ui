### Rotating dimension

- **Chosen:** Performance
- **Why:** the load-bearing risk per the plan is the **pan/zoom canvas + momentum + virtualization** interaction â€” the one place a Gantt degrades.
- **Verified?** yes (reasoned + build-verified; no profiling harness)
- **Notes:** Row virtualization (`@tanstack/react-virtual`, threshold 60) is **orthogonal** to horizontal pan/zoom (a pure `pxPerMs`/`origin` transform on bar `left`s) â€” they don't fight: the virtualizer owns vertical scroll only; the gutter mirrors `scrollTop` imperatively (no extra re-render). Pan/zoom recompute bar positions for **visible rows only** (~20), so frame cost is bounded regardless of total task count. Momentum is a single rAF decay loop, cancelled on `pointerdown`. Wheel zoom uses a **non-passive native listener** (the `story-composer-01` trap, handled). One accepted cost: the context value is rebuilt each Root render, so all parts re-render on pan (intended) and on the 60s color tick (cheap). No virtualization of *bars within a row* (one bar per row, so unnecessary).
