## Risk + verification notes

- **A4 collision detection** — `pointerWithin` for item drags (target = whatever's under the cursor), branching to `closestCenter` over **columns only** for column reorder, with `rectIntersection` as a fling fallback. Verified the column-reorder branch still requires `activeData.kind === "column"`, so an item-over-column can't trigger reorder and vice-versa.
- **A4 drop routing** — `overData.kind === "column"` now joins the `cell`/`collapsed-column` branch (append at end of column); reached only for item drags. The new path still flows through `canDrop` (`acceptsRendererIds` rejection preserved) and the same-column index-shift / no-op logic unchanged.
- **A4 cell fill** — gated on single-lane (`!hasSwimlanes`): inner wrapper `min-h-full`, lane wrapper `flex-1`, cell `grow → flex-1`. Swimlane stacks keep content-sized lanes so multi-lane drops still target the correct lane.
- **A5** — non-passive `addEventListener("wheel", …, { passive: false })` so `preventDefault()` works (React's JSX `onWheel` is passive); yields to a not-at-edge column under the pointer; only translates `deltaY` (native `deltaX`/shift+wheel left alone).
- **A1** — gated to `dragHandle: "header"` mode; shell-mode body wrapper is plain `relative` (no negative margin), so the default shell path is visually unchanged. NB: with the todo-rich-card adapter flipping to shell-drag (B8) this defends the remaining header-mode consumers (built-in `kanban-card`/`kanban-note` + the demo's custom header renderer).
- **tsc** clean (background run, exit 0); folder **lint** clean; `validate:meta-deps` 55/55; `pnpm build` succeeds.
