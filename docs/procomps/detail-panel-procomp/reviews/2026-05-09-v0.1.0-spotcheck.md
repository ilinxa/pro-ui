## Findings (3)

### F-01 — `DetailPanelError` is both a type (in `types.ts`) and a component (in `parts/detail-panel-error.tsx`)

- **Severity:** 🔹 Low
- **Dimension:** 2 — Public API (naming clarity)
- **Location:** `src/registry/components/feedback/detail-panel/types.ts:10` (interface) + `parts/detail-panel-error.tsx` (component) + `index.ts` (re-exports the type)
- **Observed:** Two distinct symbols share the name `DetailPanelError`:
  - `interface DetailPanelError { message: string; retry?: () => void }` — the error-shape type used as `error?: DetailPanelError | null` on props.
  - `<DetailPanelError error={...} />` — internal component rendered when `error` is set.
  
  TypeScript handles the disambiguation in different scopes (the type is in `types.ts`, the component is internal in `parts/`). Consumers importing only the type via index.ts get the right thing. But IDE symbol-search for "DetailPanelError" surfaces both — and the type re-export shadows the conceptual identity ("is DetailPanelError the error data, or the error UI?"). Cognitively confusing.
- **Suggested fix:** Rename the type to `DetailPanelErrorState` or `DetailPanelErrorInfo` (clearer intent — describes the data, not the renderer). Update the prop type signature + index.ts re-export. Producer-side breaking change (consumers using `DetailPanelError` as the type need to migrate); minimize impact by aliasing for one minor version: `export type DetailPanelErrorState = DetailPanelErrorInfo; export type DetailPanelError = DetailPanelErrorInfo /* @deprecated, use DetailPanelErrorState */`. v0.2 candidate.

### F-02 — `ariaLabel?: string` is optional, but `<div role="region">` requires an accessible name per WAI-ARIA

- **Severity:** 🔸 Medium
- **Dimension:** 2 — Public API (and Dim 7 — Accessibility)
- **Location:** `src/registry/components/feedback/detail-panel/types.ts:40` + `detail-panel.tsx:136-138`
- **Observed:** The root renders `<div role="region" aria-label={ariaLabel}>`. When `ariaLabel` is undefined, JSX skips the attribute, leaving a `<div role="region">` with no accessible name. Per WAI-ARIA: *"region role requires an aria-label or aria-labelledby."* Without one, SR users hear "region" without knowing what region. The current behavior is silently non-conformant.
- **Suggested fix:** Either (a) make `ariaLabel` required: `ariaLabel: string;` (breaking; principled), OR (b) provide a sensible default via a `labels` prop (consistent with the rest of the library): `labels?: { region?: string }` defaulting to `"Detail panel"`. Option (b) is the i18n-friendly fix and matches the pattern used by event-card-01 / newsletter-card-01 / filter-stack. The default English string can be stripped from the bundle if the consumer overrides via `labels`.

### F-03 — `DetailPanelEmptyState` exported as sibling but not as `DetailPanel.Empty` compound child

- **Severity:** 🔹 Low
- **Dimension:** 2 — Public API (compound-API consistency)
- **Location:** `src/registry/components/feedback/detail-panel/detail-panel.tsx:163` (sibling export) + `:159-161` (compound attachments)
- **Observed:** The compound API attaches `Header` / `Body` / `Actions` as static children (`DetailPanel.Header = DetailPanelHeader` etc.). `DetailPanelEmptyState` is exported as a separate named sibling — but conceptually it's the empty-state slot's complement, comparable to the other compound parts. A consumer using the compound idiom does:
  ```tsx
  <DetailPanel selection={sel}>
    <DetailPanel.Header>...</DetailPanel.Header>
    <DetailPanel.Body>...</DetailPanel.Body>
    <DetailPanel.Actions>...</DetailPanel.Actions>
  </DetailPanel>
  ```
  ...and to override the empty state has to import `DetailPanelEmptyState` separately and pass via the `emptyState` prop. Inconsistent. Per Dim 14 review-guide: *"A consumer can guess the API shape of a new component after seeing two siblings."* Compound children should follow one mental model.
- **Suggested fix:** Add `DetailPanel.Empty = DetailPanelEmptyState` to the compound attachments. v0.1.x non-breaking — sibling export stays for backward compatibility; compound shorthand is additive. One-line change.
