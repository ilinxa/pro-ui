## Findings (1)

### F-01 — `align="end"` (right-aligned) variant not demonstrated; consumers learn 2 of 3 alignment modes from demo

- **Severity:** 🔹 Low
- **Dimension:** 4 — Demo + usage
- **Location:** `src/registry/components/data/article-meta-01/demo.tsx` (5 sub-tabs, none with `align="end"`)
- **Observed:** `meta.features` lists *"3 alignment modes (start / center / end)"* and `types.ts` defines `align?: "start" | "center" | "end"` (default `"start"`). The demo exercises `align="start"` (default tab implicit) and `align="center"` (centered tab) but does NOT demonstrate `align="end"`. A consumer browsing the demo for layout patterns will discover only 2 of 3 alignment modes; they'd need to read types.ts to find `"end"`. Trivially fixable; the omission is conspicuous because both other modes ARE shown.
- **Suggested fix:** Either (a) add a 6th sub-tab "Right-aligned" with `align="end"` (e.g., for a comment / forum-thread context where meta is right-aligned), OR (b) repurpose the "Centered" tab into "Alignment" with three side-by-side rows (start / center / end) shown together. Option (a) is the smaller diff.
