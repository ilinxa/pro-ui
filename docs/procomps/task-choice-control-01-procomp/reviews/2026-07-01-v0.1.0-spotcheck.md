### Rotating dimension

- **Chosen:** **Copy / never-forced + no-penalty** (the hard checks, per plan §6/§12.10).
- **Why:** coercion-creep is the one risk that fails the whole element — it lives almost entirely in wording, color, and motion on the release path.
- **Verified?** yes (code + token inspection; live walkthrough is F-02)
- **Notes:**
  - **No mandatory state exists in the type system.** No `disabled`/`required`/`mustAssign` prop, no "you must choose" branch. The only non-interactive path is the opt-in `readOnly` (default `false`). Choice is the default (success criterion #2). ✅
  - **Release folds into reassign — one chokepoint (D10).** Release calls `onAssigneeChange(undefined)`; there is **no** `onRelease` to drift. ✅
  - **No penalty signal on release (hard ban list, §6):** the Release `<Button variant="ghost" className="text-muted-foreground">` — **never** `destructive`, never red. Copy is **"Release"** — never "Drop"/"Abandon"/"Unassign". No ✕-on-a-person glyph (a neutral `UserRoundPlus` for reassign, plain text for Release). The `aria-live` sentence is neutral ("Open for anyone" / "Assigned to {name}" / "Unassigned") — never "X dropped this". ✅
  - **"Open for anyone" is a friendly invite** — `Switch` + "🙌 Open for anyone", signal-lime when on, no alert treatment (D9). ✅
  - **Team-scoped:** the reassign picker lists only `members` (this team); no inter-team/public surface (D-08). ✅

**Single-unit-vs-compound:** correctly a single-unit control — flat à-la-carte parts (`OpenForAnyoneToggle` / `ClaimButton` / `AssigneeChip`) under a logic-free assembly, **no Root, no context, no `hooks/`** (the only internal state — `pickerOpen` — lives in `AssigneeChip`). The assembly's one non-trivial act (`resolveTaskChoiceState` + branch) is reproducible by hand (the resolver is exported). Subset paths proven in the demo's "À-la-carte" section.

**SSR-safety:** fully controlled (no data state); no `Date`/random during render; `pickerOpen` inits `false` on both server + client. No hydration surface.

## Intentional deviations from the plan (loud, not silent)

1. **`isOpenFlagVisible` helper dropped.** The plan (§4.1/§5) listed an always-`true` `isOpenFlagVisible(v)` export. It was never called (the assembly always mounts the toggle directly), and an unused param tripped `no-unused-vars`. Removed as dead code; the "open-flag always visible" intent is enforced structurally in the assembly + documented in a `state.ts` comment. `resolveTaskChoiceState` remains the exported resolver.
2. **Reassign picker uses Command's built-in filter** (default `shouldFilter`), with `CommandItem value={displayName}` so typing filters by name — rather than the author-picker's `shouldFilter={false}` + manual async (this list is synchronous + bounded to one team). `cmdk` is backend-agnostic, so no divergence risk from this choice.
