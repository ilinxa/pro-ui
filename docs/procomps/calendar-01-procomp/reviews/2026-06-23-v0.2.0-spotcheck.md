#### F-00 вЂ” Fixed in-review (3-pass adversarial review, 2026-06-23)

The pre-ship review surfaced one real bug + a set of polish items, **all fixed before this ship**:
- **(was вљ пёЏ High) Dead Edit/Enter under `showInspector`** вЂ” the inspector keyed the editor off *selection*, not `editingId`, so context-menu "EditвЂ¦" / `Enter` on an unselected event were no-ops. Fix: `openEditor` now also `select`s (and editor/rename are mutually exclusive).
- **(was рџ”ё) Keyboard all-day resize off-by-one + corrupt-window** вЂ” rewrote all-day resize to the exclusive-end covered-days math (matches the drag grip; clamped в‰Ґ 1 day); milestones are no longer resizable.
- **(was рџ”№) Menu Cut data-loss** in insecure contexts (deleted without copying) вЂ” now guarded on `navigator.clipboard`.
- **(was рџ”№) Clipboard hijack** while an editor/rename/composer is open вЂ” handlers now bail (native copy/paste).
- **(was рџ”№) Leftovers/honesty** вЂ” dead `onTaskClick` prop removed; `onExternalDrop`/`onDateClick` JSDoc corrected; editor overlay now manages initial focus.
- Overlays aligned to gantt's `GanttEditPopover` (scrim/`aria` parity; an improvement: shared `EventEditorPanel` reused by the inspector).
