### F-02 (Low, v0.3.1) — `Live` badge + sub-line might duplicate visually

The Live badge (in `news-badges.tsx`) renders a pulsing dot + "Live" pill. The `LiveUpdateLine` part (used in featured/large/medium/list) ALSO renders a pulsing dot + "Updated 3m ago · 14 updates" text. When both fire at once (always, since both gate on `item.isLive`), the visual has two pulsing dots within ~3rem of each other. Not broken, but unnecessarily noisy.

**Suggested fix:** drop the pulsing dot from `LiveUpdateLine` and rely on the Live badge's dot to carry the "live" semantic; keep the radio icon in the sub-line.

**Owner:** author · **Target:** v0.3.1.

### F-03 (Low, v0.3.1) — Featured variant author byline + string author can double-render

Inside `featured.tsx`, the author cluster renders `<NewsAuthorByline>` when `authorEntity` is set AND, separately, renders the string `<User /> {item.author}` "when no structured authorEntity." The condition `!item.authorEntity && item.author` correctly guards the string path. But IF a consumer sets BOTH `authorEntity` AND the legacy `author` string (which is allowed per Q-P43 soft-compat), only the structured one renders — correct, but the legacy field could surprise host developers expecting both to render.

Not a bug; potential surprise. Documentation in guide.md already explains soft-compat. No fix needed — flagging for clarity.

**Owner:** author · **Target:** confirmed working as designed.

### F-04 (Low, v0.4) — `openKebab` handle method is a no-op stub

`content-card-news-01.tsx` ships the `openKebab` handle method but it's a no-op stub. The per-variant `<NewsKebab>` owns the actual `<DropdownMenu>` trigger ref; the root doesn't currently have a way to programmatically reach into it. The 10 other handle methods all work; only `openKebab` is symbolic.

**Suggested fix:** expose a `useImperativeHandle`-style trigger ref from `NewsKebab` upward, OR use a `useState`-driven `open` prop on `DropdownMenu`. Mid-effort; can defer.

**Owner:** author · **Target:** v0.4 (or drop the method from the Handle interface if no consumer asks for it).
