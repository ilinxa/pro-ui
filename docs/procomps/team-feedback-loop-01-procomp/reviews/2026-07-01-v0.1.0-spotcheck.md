### Rotating dimension

- **Chosen:** Dim — **Accessibility / D-10 non-blocking** (per plan §17).
- **Why:** the non-blocking guarantee + reduced-motion branch + team-subject copy are where a celebration overlay goes wrong; this is the component's entire risk surface.
- **Verified?** yes (by code + built-output inspection; live browser walkthrough is F-02)
- **Notes:**
  - **Non-blocking (D-10) — verified structurally.** `CelebrationOverlay` wrapper is `pointer-events-none`; the card is `pointer-events-none`; the **only** `pointer-events-auto` element is the skip `<button>`. No `.focus()` call anywhere, no focus-lock primitive, no work-halting scrim — the band is a bottom-edge flourish, not a centered modal. `role="status"` + `aria-live="polite"` (announces without interrupting). Esc is an additive `document` keydown listener (no preventDefault/trap), removed on close.
  - **Timer clamp** — `clampDuration` = `min(max(ms ?? 800, 200), 999)`; a `celebrationDurationMs={5000}` is clamped to 999 by construction → the "modal that lingers" failure mode is impossible.
  - **Reduced-motion** — a real branch: `reveal-up` dropped, `canvas-confetti` never mounted (gated on `!reducedMotion`) + `disableForReducedMotion` belt-and-suspenders; still time-boxed + skippable. Read via `useSyncExternalStore` (SSR-safe, reactive to OS toggles).
  - **Team-subject copy** — every string in `dummy-data.ts` + demo + overlay is team-scoped ("Your team…"); no individual name, no per-member call-out.
  - **Confetti lazy boundary — proven in the built output.** `canvas-confetti` is isolated in a dedicated **10.6KB chunk** with **0** overlay (`Dismiss celebration`) and **0** reducer (`nudge-rearm`) markers; the component code lives in separate chunks. Default CSS path + `enableConfetti={false}` never load it.

**SSR-safety:** `current` starts null (overlay renders nothing on first paint); `reducedMotion` server-snapshot = false; timer/Esc/matchMedia all client-only. No animation on SSR/first paint. No `set-state-in-effect` (reduced-motion via `useSyncExternalStore`, not matchMedia-into-state).

**Reducer correctness:** both trigger paths (controlled `event` effect + imperative `celebrate()`) funnel into the same `open`/`close` actions; single `current` slot → overlays cannot stack; skip vs auto-dismiss never double-fire `onCelebrationDismiss` (skip clears the pending timer); host-driven `event=null` closes silently (controlled-echo).

## Intentional deviations from the plan (loud, not silent)

1. **Coalesce window (§8.1)** — implemented the plan's **sanctioned fallback** ("every `open` bumps `epoch`") rather than the ~50 ms leading-edge coalesce. Still stack-proof (single slot); a rapid burst restarts the < 1s timer per event instead of coalescing. The plan explicitly allows this ("if review finds it over-engineered, the fallback is…"). Leading-edge coalesce → v0.2 if flicker is observed.
2. **Reduced-motion via `useSyncExternalStore`** instead of the plan's `matchMedia`-into-state-in-effect — avoids the repo's `set-state-in-effect` lint error; idiomatic + SSR-safe. Same behavior.
3. **`CelebrationOverlayProps` gained a `children` slot** (hosts the confetti burst) beyond the plan's listed props — minor, keeps the overlay dumb while the Tier-B part gates the lazy confetti.
4. **Assembly props type** named `TeamFeedbackLoop01Props` (library convention + scaffolder), not the plan's sketch `TeamFeedbackLoopProps`.
