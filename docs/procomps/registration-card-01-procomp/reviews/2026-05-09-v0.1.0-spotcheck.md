## Findings (2)

### F-01 — `spotsLeftSuffix: "spots left"` produces ungrammatical "1 spots left" — RECURRING bug from event-card-01 F-01 (s11)

- **Severity:** 🔸 Medium
- **Dimension:** 11 — UI copy / text (and Dim 7 — Accessibility, since SR users hear the broken text)
- **Location:** `src/registry/components/data/registration-card-01/types.ts:10` + `registration-card-01.tsx:122-123`
- **Observed:** `${spotsLeft} ${labels.spotsLeftSuffix}` → "5 spots left" (correct) BUT also "1 spots left" (wrong). Per Dim 11 review-guide bad signal: *"Pluralization correct ('1 item' / '2 items'), no '1 items'."* This is the **2nd occurrence of the exact same bug shape**, after event-card-01 F-01 (session 11) — both components ship `spotsLeftSuffix: "spots left"` as a default; both produce "1 spots left" at the boundary. Common when capacity is filling (last spot countdown).
- **Suggested fix:** Bundle with event-card-01 F-01 as a single v0.1.1 patch. Both components would gain `formatSpotsLeft?: (count: number) => string` callback prop with default using `Intl.PluralRules`. One cohesive fix across the family.

### F-02 — `${capacity} ${capacitySuffix}` produces awkward "500 capacity" without a connector

- **Severity:** 🔹 Low
- **Dimension:** 11 — UI copy / text (clarity)
- **Location:** `src/registry/components/data/registration-card-01/types.ts:16` + `registration-card-01.tsx:136-138`
- **Observed:** Bottom row renders two captions side-by-side:
  - Left: `${registered} ${labels.registeredSuffix}` → "423 registered"
  - Right: `${capacity} ${labels.capacitySuffix}` → "500 capacity"
  
  Visual layout (left-vs-right justified) implies the relationship "423 of 500 registered." But SR users hear them sequentially: "423 registered, 500 capacity" — without the "of" connector, the second reads as a separate fact rather than the denominator. Per Dim 11 review-guide: *"Reading the component's surface aloud sounds professional."* "500 capacity" reads as a stub (where's the connector? — should be "of 500" or "out of 500"). Sighted users decode the layout; SR users miss the relationship.
- **Suggested fix:** Either (a) change `capacitySuffix` default from `"capacity"` to `"of capacity"` (concatenates as "500 of capacity" — still awkward) — or (b) restructure the row to a single sentence: `"${registered} of ${capacity} ${labels.registeredSuffix}"` → "423 of 500 registered". Option (b) is cleaner SR experience but visually different. Or (c) keep the current visual but add a `aria-label` on the row consolidating: `aria-label="${registered} of ${capacity} registered"`. Option (c) is the lowest-friction fix.
