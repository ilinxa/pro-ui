## What changed

The `post` content type now authors **multi-media** (Instagram-feed semantics) via a
new **`mediaCarouselSlot`** kind backed by `media-carousel-editor-01`. Strictly
additive: `news` keeps the single `mediaSlot`. New surface = `MediaCarouselSlotConfig`
/ `MediaCarouselSlotValue` / `MediaCarouselItemRef` + a `media-carousel-substrate`
that bridges the carousel's blob-carrying items to the JSON-clean draft, + a shell
`CarouselLiveCacheContext` that preserves live items across step navigation.

The multi-blob **upload-at-publish** is deferred with the (already-deferred) post
backend adapter — authoring is live; publish surfaces the existing "no adapter" error.

## Adversarial review → findings fixed

A 3-agent review (code-correctness / UX+a11y / architecture) was run against the
carousel + integration. **24 findings surfaced; all blockers + highs + most
mediums fixed this pass.** Carousel changes ship as **v0.1.1**; integration as
content-composer **v0.2.0**.

**Fixed (carousel v0.1.1):** object-URL leak on controlled value-swap (orphan-revoke
effect) · intake race + cap (synchronous `itemsRef` + cap in `addItems`) · loading
state during ingest · in-UI error/max surface · screen-reader announcements +
dnd-kit announcements · export busy/`aria-busy` + error surface · re-edit
double-overlay (`initialSource` only when no `editorState`) · video a11y dead-end
(static caption, not a disabled button) · 24px hit targets · video poster
(`playsInline` + seek) · drag-over on add-more · empty-main fallback · rail
read-only cue · `nextId` collision prefix · dead `"max-items"` error kind removed ·
`useCarouselState` exported.

**Fixed (content-composer v0.2.0):** **silent local-media loss across step nav** →
`CarouselLiveCacheContext` (shell keeps live items; carousel runs
`revokeOnUnmount={false}`; root revokes cached blob URLs on unmount) · dead
`carouselRef` removed · new public types exported from barrel · `getIsDirty` baseline
(was `length>0`) · `reconstruct` warns on dropped local refs + doesn't persist
unrecoverable `editorState` · `assertNever` exhaustiveness guards on the gate
switches.

## Findings (remaining — none blocking)

#### F-01 — Multi-blob upload-at-publish deferred (N-egress is `mediaSlot`-shaped)

- **Severity:** 🔸 Medium (deferred, documented)
- **Dimension:** Composition integrity
- **Observed:** `captureMediaBlob` / `uploadHero` / `SlotHandle.export?` are single-blob (`mediaSlot`-only). The carousel step authors + persists structure, but durable upload of N blobs isn't wired — it rides with the post backend adapter (also deferred).
- **Plan:** v0.3 "post backend" cohort, co-owned with the `post-content-item` adapter. Generalize `SlotHandle.export?` → a plural egress (`MediaUploadUnit[]`) at that time. Tracked.

#### F-02 — Cross-RELOAD durability of local items

- **Severity:** 🔹 Low (architectural seam, documented)
- **Observed:** step-nav is now lossless (live cache), but a full page reload still can't restore un-uploaded local blobs (a blob can't live in the JSON draft). Surfaces a dev warning; rides with the upload backend.

#### F-03 — Minor polish deferred

- **Severity:** 🔹 Low
- edit↔preview swap has no motion-safe transition (currently instant — no motion violation); `aria-current="true"` vs `"step"`; lime focus-ring contrast not formally measured (tokens are correct).
