### Rotating dimension — Composition integrity (LOCKED)

- **Chosen:** Composition integrity.
- **Why:** this shell composes three heterogeneous substrates (json-form / Plate / media-editor) behind one uniform `SlotHandle`; prop-flow correctness, no leaked internals, and correct dirty/validity aggregation across the asymmetric slots are the load-bearing risk.
- **Verified?** yes.
- **Notes:**
  - **Prop flow:** the root threads `{ slotConfig, value, onChange, ctx, handleRef }` to each substrate via `SlotMount`; `onChange` wraps back into the discriminated `ComposerStepValue` at the root. One documented dispatch-boundary cast at the `SlotMount` call site (`SlotHandle` is invariant in `TValue` via `loadValue`) — runtime-correct, localized. ✅
  - **No prop-drilling hacks:** state is fully lifted to the root; substrates are controlled; `useComposerState` owns the draft + FSM. ✅
  - **Dirty aggregation:** refined to **draft-level** (`draft !== savedDraft`) rather than the plan's handle-aggregate — single correct signal under single-step mounting AND structurally averts the Plate autosave-loop (#1 trap). The body baseline (`use-body-dirty`) stays only for the headless `SlotHandle.getIsDirty` contract. ✅
  - **Gate correctness:** value-based gating against the always-current draft (per-mutation `onChange`) — required because news has 3 metadataFields steps and only the active one mounts; the plan's handle-keyed-by-slot publish-all would read stale handles. The active metadata step still gets full json-form validation. ✅ (see F-02 for the non-active-step nuance)
  - **Substrate boundary:** the uniform `SlotHandle` carries an optional media-only `export?` (see F-04) — a minor leak, documented + duck-typed.

## Findings (5)

#### F-01 — Consumer-tsc smoke pending (second inter-procomp install)

- **Severity:** 🔸 Medium
- **Dimension:** Verification
- **Location:** path-b smoke (C18)
- **Observed:** the F-cross-11 path-b consumer-tsc smoke runs at C18 against the deployed artifact. This is the SECOND inter-procomp registry-dep install in the library (after story-composer-01→media-editor-01); the cross-procomp `@/registry/.../types` + deep-lib (`json-form/lib/condition-evaluator`, `/lib/path`, `/lib/define-field-renderer`) imports are F-cross-13 / F-S1 risk surfaces.
- **Suggested fix:** run the smoke at C18; expect + patch sub-traps in a v0.1.1 (the 4-ship pattern). The cross-procomp `/types` reference is centralized in `types.ts` (one place to fix if the rewriter mangles it). Owner: author. Target: v0.1.1.

#### F-02 — Non-active-step metadata gate is required-presence only

- **Severity:** 🔹 Low
- **Dimension:** Composition integrity
- **Location:** `lib/gates.ts:evaluateMetadataValue` / `evaluateStep`
- **Observed:** for the ACTIVE metadata step the gate calls `handle.validate()` (full json-form trigger + isValid — catches pattern/custom validators). For NON-active steps (publish-all over unmounted steps) it falls back to a required-presence check over stored values. A field that is present-but-pattern-invalid on a non-active step could pass the publish gate (it was fully validated when the user last left it forward, so this is narrow).
- **Suggested fix:** for full parity, validate non-active metadata values against the json-form Zod schema headlessly (reach `json-form/lib/schema-compiler`), or mount all steps hidden (rejected — media camera-permission risk). Owner: author. Target: v0.1.x (if a config surfaces it).

#### F-03 — Inline body-image upload unwired

- **Severity:** 🔹 Low
- **Dimension:** Composition integrity
- **Location:** `parts/body-substrate-plate.tsx` (`onImageUpload` optional, unwired)
- **Observed:** the composer `uploader` is `(blob, ExportMetadata) => …` (media-export-specific). Fabricating an `ExportMetadata` (textOverlays/stickers/adjustments/crop) for an inline article image is semantically wrong, so Plate falls back to its URL-prompt image insertion in v0.1.
- **Suggested fix:** add a dedicated `bodyImageUploader?: (file) => Promise<{ src }>` prop and thread it. Owner: author. Target: v0.1.1.

#### F-04 — `export?` leaks into the uniform `SlotHandle`

- **Severity:** 🔹 Low
- **Dimension:** Composition integrity
- **Location:** `types.ts:SlotHandle.export?`
- **Observed:** the otherwise-uniform `SlotHandle<TValue>` carries an optional media-only `export?` so the shell can pull the hero blob at publish. The shell duck-types it; other slots omit it. It's a minor abstraction leak into the generic interface.
- **Suggested fix:** introduce a `MediaSlotHandle extends SlotHandle<MediaSlotValue>` subtype and narrow at the media call site. Owner: author. Target: v0.2.0.

#### F-05 — Video re-edit blob re-attach deferred

- **Severity:** 🔹 Low
- **Dimension:** Composition integrity
- **Location:** `parts/media-substrate.tsx:loadValue`
- **Observed:** `loadValue` restores photo via `editorState.imageSrc` (videoBlob nulled). A VIDEO draft would restore with an empty source — the locked invariant (re-attach the blob from the shell Map / re-fetch from `exportedUrl` before `loadState`) is not yet wired. News v0.1 is photo-only, so this is unexercised.
- **Suggested fix:** wire video-blob re-attach alongside the first video config. Owner: author. Target: v0.2.0 (with the video config / post ship).
