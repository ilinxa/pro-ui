### Rotating dimension

- **Chosen:** **Copy / never-forced framing** (with Accessibility folded in — the never-forced rule lives in both).
- **Why:** the whole component exists to make refusing cost-free + obvious (system §5.2, the load-bearing rule). Copy + a11y are exactly where a cooperative-challenge surface drifts into "you failed / you're locked out."
- **Verified?** yes (by code + design-token inspection; live browser walkthrough is F-02)
- **Notes:**
  - **Opted-out is neutral + joinable, not failure.** `isJoinable` → the card renders full-opacity with `border-border` + `bg-card` (never `--destructive`, never a dimmed/disabled treatment); the header shows an **"Optional"** badge (`variant="outline"`), the reward + goal are shown as *available*, and a **prominent primary** `Join this challenge` button (`variant="default"`, signal-lime) reads as an invitation. Verified in the code + the demo's "Joinable" section.
  - **Leaving is one click, no guilt.** When `isActive`, the control flips to `leaveLabel` with `variant="outline"` (plain) — **no** confirmation dialog, **no** "abandon?" copy, **no** warning tone. There is no `required`/`locked`/`mandatory` prop anywhere in the type surface.
  - **`noPenaltyHint` visible in both states**, `aria-describedby`-associated with the control ("Optional — no penalty for sitting this one out").
  - **Collective progress only.** `ChallengeProgressMeter` renders one bar + one `current / target` count. **No** per-member row/ranking/"you still need to" framing exists; `TeamMemberStack` is identity-only (no per-avatar state) — the only member-level surface, and it's a pile, not a progress display.
  - **Whole-team reward.** `ChallengeRewardChip` copy is *"The team earns:"* → *"The team earned:"* (`earned`). No first-person path, no per-member reward field in the type.
  - **Done = lightweight inline ack, no modal.** `isComplete` → a "Completed together" pill (glyph + text, not color-alone) with a `.reveal-up` micro-entrance (globally reduced-motion-safe) + `aria-live="polite"` on the pill only (announced once, no focus steal). The heavy celebration is E6's job — **not built here**; no import of `team-feedback-loop-01`.
  - **A11y:** card is `role="group"` + `aria-label`; opt-in is a labelled `<button>` with `aria-pressed`; meter is `role="progressbar"` with `aria-valuetext` = "current of target" via Radix `getValueLabel`; member stack is one `aria-label`, avatars decorative.

## Intentional deviations from the plan (loud, not silent)

1. **Progress meter aria semantics.** The plan (§10) specced `aria-valuenow={current}` / `aria-valuemax={target}`. The shipped shadcn `progress` wrapper drives its **visual fill from a 0–100 `value`** (it computes `translateX` from `value`, not from `value/max`), so `value` must be the percentage. To still convey "3 of 5" to AT, the meter passes Radix's `getValueLabel` → `aria-valuetext="3 of 5"` (which screen readers read in preference to the numeric value). Net a11y outcome is the plan's intent ("AT reads 3 of 5, not a bare %"); the underlying `aria-valuenow` is the percentage. Acceptable + documented.
2. **Earned-ack lives in the Header**, not a separate export. The plan's §4.1 tier inventory has no earned-ack part; §6.4 requires a "Completed together" inline ack. Folded into `CooperativeChallengeHeader` (the context-aware part that already renders the state indicator) rather than adding an un-planned export — keeps the locked export surface exact.
3. **Copy defaults centralized** in `lib/derive.ts` (`DEFAULT_JOIN_LABEL` / `DEFAULT_LEAVE_LABEL` / `DEFAULT_NO_PENALTY_HINT`) so the Root and the context-free `OptInToggle` can't drift — one source of truth for the never-forced strings.

#### F-03 — Meter `aria-valuenow` is the percentage, not `current`

- **Severity:** 🔹 Low
- **Dimension:** Accessibility
- **Location:** `parts/cooperative-challenge-progress.tsx`
- **Observed:** Because the shadcn `progress` wrapper fills from a 0–100 `value`, `aria-valuenow` is the percentage; the "3 of 5" reading is delivered via `aria-valuetext` (`getValueLabel`). AT that honors `aria-valuetext` (all major SRs) reads it correctly; the raw numeric value is the pct.
- **Suggested fix:** Acceptable as-is (WAI-ARIA prefers `aria-valuetext`). If ever undesirable, a bespoke meter (not the shared primitive) could set `aria-valuenow={current}`/`max={target}` directly — v0.2 only if a real need surfaces.
