## Notes

Markdown-editor is the heaviest pro-component in the registry (~180KB ceiling; 28 files; CM6 substrate; per-instance Marked; runtime-updatable wikilink decoration via StateField). The implementation is **structurally faithful to the plan to a remarkable degree** — fewer plan-vs-code drift findings than workspace despite the plan being 4× larger.

The findings target the ecosystem (F-cross-08, F-cross-01, phantom dep declaration) more than the editor's substrate work. F-03 (phantom `radix-ui`) is a subtle but real meta-correctness issue that would silently bloat consumer installs.

The session-5 fresh smoke run captured **F-cross-09** (shadcn CLI regression). Workspace + markdown-editor reviews jointly surface this; recommend pinning smoke harness to `shadcn@4.6.0` until the regression is understood.
