### Rotating dimension

- **Chosen:** Dim — **Robustness** (per plan §14).
- **Why:** the SSR-safe diff seam + the lazy award boundary are the whole build's risk surface — the same set-state-in-effect / hydration-flash class calendar-01 + rich-card solved, plus a real lazy chunk to prove.
- **Verified?** yes
- **Notes:**
  - **SSR-safe award diff** — implemented via React's "storing information from previous renders" pattern (setState **during render**), NOT a `mounted` effect flag. `prevBadges` seeds from the first render's `badges`, so the initial/hydration render has `prev === current` → `newAwards` empty → nothing animates on load. A later controlled `badges` update flips exactly one award set. This is a **deliberate improvement over the plan's literal `prevBadgesRef` + `mounted` flag**: the plan's `useEffect(() => setMounted(true))` would trip the repo's `set-state-in-effect` lint rule (which errors, seen live on other files this session); the chosen pattern avoids both set-state-in-effect AND ref-during-render while staying hydration-safe.
  - **Lazy boundary proven in the built output** — `BadgeAwardOverlay` compiles to a dedicated **1517-byte chunk** containing the spark burst but **none** of the grid (`flex flex-wrap gap-4` → 0) or token (`rounded-2xl transition-colors` → 0) code, which live in separate chunks. The bare-token path and `animateAward={false}` never reference the lazy import → chunk never loads.
  - **Reduced-motion** — the pop is `motion-safe:`-gated; the burst span is `motion-reduce:hidden`; `reveal-up` is reduced-motion-gated in globals.css. All three skip to settled.
  - **Deterministic dates** — `formatAwardedDate` uses UTC parts + a fixed month array (no `Date.now()`, no locale) → the awarded date reads identically server + client. The tooltip content is portaled/on-open, so it isn't in SSR HTML either.

**Cooperative-only check (system §5.3 / D-08):** no `members`, no per-person attribution, no rank/comparison; the header count is "N of this team's M"; `avatar` deliberately unused. No leak. ✅

**D-16 (celebration ownership):** `animateAward` JSDoc + the guide loudly document setting it `false` to defer to `team-feedback-loop-01`; the component carries no hook into E6. ✅

#### F-03 — Non-interactive earned badge: awarded-date tooltip is hover-only

- **Severity:** 🔹 Low
- **Dimension:** Accessibility
- **Location:** `parts/team-milestone-badge.tsx` (non-interactive `role="img"` span as `TooltipTrigger`)
- **Observed:** When `onOpen` is absent the token is a non-focusable span, so the awarded-date tooltip opens on hover but not keyboard focus. Mitigated: the date is already in the token's `aria-label`, so AT users don't lose it.
- **Suggested fix:** v0.1.1 — optionally make the non-interactive token `tabIndex={0}` when it has a tooltip, or accept the aria-label as the AT path (documented).

#### F-04 — Each shelf is a labelled landmark region

- **Severity:** 🔹 Low
- **Dimension:** Accessibility
- **Location:** `parts/team-trophy-shelf-root.tsx` (`<section aria-label>`)
- **Observed:** A labelled `<section>` is a `region` landmark; multiple shelves on one page (e.g. the demo) produce several regions named "Team trophies".
- **Suggested fix:** v0.1.1 — consider a non-landmark wrapper by default, or document overriding `aria-label` per shelf. Consumers can already override.
