### Rotating dimension

- **Chosen:** Dimension 2 — Public API
- **Why:** This patch adds two new optional `WorkspaceProps` (`onError`, `cardStackItemHeight`) + a new internal `<SplitDivider>` prop (`onKeyResize`) + a new exported hook (`useResizeKeyboard`). The rotating dim is exactly the surface most likely to surface drift between intent and code.
- **Verified?** **yes**
- **Notes:**
  - `onError?: (errors: string[]) => void` — purely additive; default behavior is `console.error` (preserved); callback fires via a `useRef` to avoid re-running the validate effect on prop-identity change. **No public-shape regression.**
  - `cardStackItemHeight?: number` — defaults to 320 (the prior hardcoded value); threaded through `<CardStack>` → `<StackedCard>` via the new `itemHeight` prop. Backwards-compatible.
  - `<SplitDivider onKeyResize>` is component-internal (not exported in `index.ts`). Outside consumers can't reach it; safe to evolve.
  - `useResizeKeyboard` is exported alongside `useKeyboardActions` from `./hooks/use-keyboard-actions`. Workspace itself calls it; consumers extracting the hook in isolation would need to provide `leaves` / `dividers` / `renderedTree` themselves — not currently a documented surface but the export keeps the option open.
  - `AreaTree` discriminator is **unchanged** in v0.1.2. The `stack` kind addition (B-2) is the v0.2.0 breaking-change story.
