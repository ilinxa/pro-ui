### Rotating dimension

- **Chosen:** Dim 11 — UI copy / text
- **Why:** newsletter-card-01 is a marketing card whose primary function is asking visitors to commit to a recurring touchpoint. Polish on visible strings (title, body, button, success/error messages, placeholder, ARIA) carries disproportionate weight for perceived quality. Migration-from-kasder also raises the question of whether the English defaults were thoughtfully translated rather than mechanically translated.
- **Verified?** yes (with 1 small finding)
- **Notes:**
  - **All visible strings centralized in `DEFAULT_LABELS`** (`types.ts:28-36`) — title / body / placeholder / emailLabel / button / successMessage / errorMessage. ✓ matches Dim 11 review-guide *"keep visible strings near the top"* (i18n-readiness).
  - **Sentence case throughout** ✓ matches Dim 11 review-guide *"Sentence case, not Title Case"*: "Join our newsletter" / "Latest updates, straight to your inbox." / "Email address" / "Subscribe" / "Thanks! You're subscribed." / "Something went wrong. Please try again."
  - **Verb-first button** ("Subscribe") ✓.
  - **Celebratory success message** ("Thanks! You're subscribed.") ✓ avoids robotic "Submitted" / "OK".
  - **ARIA hygiene:**
    - `aria-busy` on button during pending ✓
    - `aria-live="polite"` on success region ✓
    - `role="alert"` on error region ✓ (forces immediate announcement vs polite-deferred)
    - `emailLabel` separate from `placeholder` for SR vs sighted users.
  - **Turkish translation in `dummy-data.ts`** (`NEWSLETTER_CARD_LABELS_TR`) — full surface translated (heading / body / placeholder / emailLabel / button / success / error). No partial translation. ✓

## Findings (1)

### F-01 — Default `placeholder: "Email address"` is generic; format-hint placeholder is more polished

- **Severity:** 🔹 Low
- **Dimension:** 11 — UI copy / text
- **Location:** `src/registry/components/marketing/newsletter-card-01/types.ts:31` (DEFAULT_LABELS.placeholder)
- **Observed:** Per the Dim 11 review-guide bad-signal: *"Placeholder text in inputs explains the format ('yyyy-mm-dd'), not 'Enter date'."* The current placeholder `"Email address"` is functional but generic — it duplicates the `emailLabel` (the aria-label) verbatim, and doesn't model the format. The polished alternative is `"you@example.com"` (or similar realistic hint), which both demonstrates the format and avoids redundant aria-label/placeholder text. Minor — a placeholder of "Email address" will not confuse anyone, but it's the kind of small polish that distinguishes a sophisticated component from a half-built one (the dimension's stated value).
- **Suggested fix:** Update `DEFAULT_LABELS.placeholder` from `"Email address"` to `"you@example.com"` (or a similar locale-neutral example). One-line change. Update Turkish dummy-data analogously (e.g., `"adiniz@ornek.com"`) — keep `emailLabel` as the descriptive aria-label.
