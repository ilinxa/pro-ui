## Findings

#### F-01 — v0.3 guide section not yet authored

- **Severity:** 🔸 Medium
- **Dimension:** Planning docs
- **Location:** `docs/procomps/story-viewer-01-procomp/story-viewer-01-procomp-guide.md`
- **Observed:** v0.3.0 ships without an updated guide section covering the comments panel + bookmark removal + DM semantic clarification. v0.2.x guide is current up to v0.2.2; v0.3 section is missing.
- **Suggested fix:** Author v0.3 guide section. **Bump target:** v0.3.1 docs (immediate follow-up).

#### F-02 — Backdrop catcher uses `<button>` with `inset-0` — semantic mismatch

- **Severity:** 🔹 Low
- **Dimension:** Public API
- **Location:** `src/registry/components/media/story-viewer-01/story-viewer-01.tsx` (commentsOpen backdrop block)
- **Observed:** The full-area click-catcher is implemented as a `<button>` to get keyboard accessibility cheaply. Semantically it's a backdrop dismisser, not a button. Screen readers will announce it as a button with the panel-close label, which is acceptable but not ideal.
- **Suggested fix:** Consider `<div role="button" tabIndex={0}>` + `onKeyDown` for Enter/Space, OR keep as-is and document the rationale. **Bump target:** v0.4.0 a11y polish.

#### F-03 — Scale transform may clip rounded corners on the scaled visual

- **Severity:** 🔹 Low
- **Dimension:** Public API (visual)
- **Location:** `src/registry/components/media/story-viewer-01/story-viewer-01.tsx` (scaling wrapper)
- **Observed:** The wrapper that scales has `origin-top` + a transform but no `overflow-hidden` on the inner that might be needed to keep the scaled visual from showing pixels outside its rounded container. Depends on inner element's own clipping; needs browser check.
- **Suggested fix:** Visual check in dev server (mobile portrait + desktop modal). If clipping shows, add `overflow-hidden` to the scaling wrapper. **Bump target:** v0.3.1 visual.

#### F-04 — `commentsOpen` auto-pause uses `setPaused` directly — bypasses any consumer paused state

- **Severity:** 🔹 Low
- **Dimension:** Public API
- **Location:** `src/registry/components/media/story-viewer-01/story-viewer-01.tsx` (commentsOpen useEffect)
- **Observed:** Opening the panel sets paused=true; closing the panel restores paused=false unconditionally. If the consumer had explicitly paused via `ref.current.setPaused(true)` and the user opened+closed the panel, the consumer's pause state is overwritten.
- **Suggested fix:** Capture pre-open state on open, restore on close. **Bump target:** v0.3.1 (low priority; edge case).

#### F-05 — Bookmark removal is breaking; v0.2.x consumers need a clean migration note

- **Severity:** 🔹 Low
- **Dimension:** Public API
- **Location:** `meta.ts` + decision file
- **Observed:** `onBookmarkStory` prop removed. TypeScript surfaces the break at compile-time, but a CHANGELOG-style migration note in the guide would help.
- **Suggested fix:** Add a "v0.2 → v0.3 migration" note in the guide. **Bump target:** v0.3.1 docs (paired with F-01).
