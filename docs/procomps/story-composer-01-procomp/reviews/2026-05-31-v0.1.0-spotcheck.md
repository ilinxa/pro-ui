### Rotating dimension тАФ **Performance** (justified)

- **Chosen:** Dim 4 тАФ Performance
- **Why:** This procomp combines four perf-sensitive substrates in one component for the first time in the library тАФ Konva multi-layer canvas with on-stage filter chains, ResizeObserver-driven stage sizing, `canvas.captureStream(30)` + MediaRecorder real-time video compositing, and XMLHttpRequest upload progress. None of these combinations exist in prior procomps.
- **Verified?** тЪая╕П partial тАФ spot-checked the hot paths; no live-device profiling.
- **Notes:**
  - Filter re-application path (`parts/composer-editor.tsx` effect at L~80) calls `node.cache()` + `node.filters([...])` + per-filter setters on every `adjustments` change. The adjust sliders fire `onValueChange` per-frame during drag without throttle тАФ see F-01.
  - Drawing extendTo (`hooks/use-drawing-stroke.ts:50`) clones the points array (`[...s.points]`) every pointer frame to trigger React re-render. For long strokes this is O(n┬▓) array allocation. See F-02.
  - `compositeVideo()` snapshots `stage.toCanvas()` per RAF frame (up to 30/s ├Ч 30s = 900 snapshots for max-duration). Expected to be slow on weak devices; acceptable for v0.1 but documented in the guide as a known cost (added during this review).
  - Built-in stickers (36 SVG data URLs, ~6KB total): no perf concern.
  - SSR boundary via React.lazy + Suspense is the right pattern for the `next/dynamic`-banned registry context.

## Findings (5)

#### F-01 тАФ Adjust-slider re-cache thrashing on drag

- **Severity:** ЁЯФ╕ Medium
- **Dimension:** Performance
- **Location:** `src/registry/components/media/story-composer-01/parts/composer-editor.tsx` тАФ `useEffect` with `[image, activeFilter, adjustments]` deps
- **Observed:** Every slider drag tick fires `onValueChange` тЖТ updates `adjustments` state тЖТ triggers the effect тЖТ `node.cache()` (re-rasterises the entire image) + applies filters. On a 4MP photo + Blur filter, that's ~20-40ms per cache call. Without throttle, drag stalls visibly mid-gesture.
- **Suggested fix (v0.1.1):** RAF-throttle the `adjustments` updates in `tool-adjust-sliders.tsx`, OR debounce the cache call inside the effect. Match the pattern from rich-card v0.4.x's slider work.

#### F-02 тАФ Drawing extendTo allocates a new points array per pointer frame

- **Severity:** ЁЯФ╣ Low
- **Dimension:** Performance
- **Location:** `src/registry/components/media/story-composer-01/hooks/use-drawing-stroke.ts:50` тАФ `setCurrentStroke({ ...s, points: [...s.points] })`
- **Observed:** Each `extendTo` clones the full points array (2 numbers added, N numbers cloned). For a 500-point stroke that's 500 cloned arrays totalling 125,000 number assignments. Real cost is small in absolute terms (microseconds) but GC pressure adds up on multi-stroke sessions.
- **Suggested fix (v0.1.x):** Bump a counter ref + use it as the React signal (or store points outside React state and use `useSyncExternalStore`). Defer if profiling doesn't show measurable drop.
