## Findings (2)

### F-01 — `daysUntilSuffix: "days left"` and `spotsLeftSuffix: "spots left"` produce ungrammatical "1 days left" / "1 spots left"

- **Severity:** 🔸 Medium
- **Dimension:** 11 — UI copy / text (and Dim 7 — Accessibility, since SR users hear the broken text)
- **Location:** `src/registry/components/data/event-card-01/types.ts:53, 58` + DEFAULT_EVENT_CARD_LABELS
- **Observed:** Two suffix-style labels concatenate to a number on render:
  - `${days} ${labels.daysUntilSuffix}` → "5 days left" (correct) BUT also "1 days left" (wrong)
  - `${spotsLeft} ${labels.spotsLeftSuffix}` → "10 spots left" (correct) BUT also "1 spots left" (wrong)
  
  Per Dim 11 review-guide bad signal: *"Pluralization correct ('1 item' / '2 items'), no '1 items'."* The current strings will produce "1 days left" / "1 spots left" for single-day / single-spot edge cases — common at the boundary (capacity countdown on the last spot, day-before-event countdown). Sighted users overlook it; SR users hear the broken text verbatim.
- **Suggested fix:** Either (a) replace the suffix-strings with formatter callbacks: `formatDaysUntil?: (count: number) => string` + `formatSpotsLeft?: (count: number) => string` — defaults use `Intl.PluralRules` for the consumer's locale. OR (b) split each into singular + plural keys: `daysUntilSuffixOne: "day left"` + `daysUntilSuffixOther: "days left"`. Option (a) handles all locales correctly (locale-specific plural rules can have more than 2 forms, e.g., Russian); option (b) is simpler but English-centric. Recommend (a) for v0.1.1 — matches the existing `formatDate?: (dateString: string) => string` callback pattern.

### F-02 — `ongoing: "Live now"` and `ongoingIndicator: "Happening now"` say the same thing in two places

- **Severity:** 🔹 Low
- **Dimension:** 11 — UI copy / text (clarity / dedup)
- **Location:** `src/registry/components/data/event-card-01/types.ts:42, 54` (both labels apply to the `ongoing` status)
- **Observed:** Both labels surface during `ongoing` status — `ongoing` is the status-badge text ("Live now") and `ongoingIndicator` is the pulsing-dot pill text ("Happening now"). Two label keys, two near-synonymous English defaults, both visible on the same card simultaneously. The visual variation (badge vs pulsing pill) is intentional per `meta.features` ("Pulsing-dot pill text on `ongoing` events"), so the dual-keys can stay — but the English defaults shouldn't read as redundant for consumers using the bare defaults. *"Live now"* + *"Happening now"* on the same card reads as repetition.
- **Suggested fix:** Either (a) align both defaults to the same string ("Live now" / "Live now" — consistent voice) and document that consumers can override one OR the other for differentiation; OR (b) make one more terse and one more descriptive (e.g., `ongoing: "Live"` + `ongoingIndicator: "Happening now"` — distinct registers); OR (c) document the dual-purpose in the guide so consumers know to override both for i18n. Option (b) is the polish fix — variation in register signals "two kinds of information", not "we forgot to dedupe".
