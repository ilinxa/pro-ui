## Findings (max 5)

#### F-01 — `history.execute()` not wrapping individual mutations

- **Severity:** 🔸 Medium
- **Dimension:** Public API (handle semantics)
- **Location:** [src/registry/components/media/media-editor-01/media-editor-01.tsx](../../../../src/registry/components/media/media-editor-01/media-editor-01.tsx) (entire mutation surface)
- **Observed:** R4 wired `useHistory` at the root and replaced the undo/redo handle stubs with `history.undo()` / `history.redo()`. Keyboard binding (Ctrl/Cmd+Z) is functional. However, no individual mutation (addTextOverlay, removeTextOverlay, addSticker, removeSticker, addDrawingStroke, setFilter, setAdjustments, setCrop) is wrapped in `history.execute(...)`, so the undo/redo stacks stay empty in practice. `history.canUndo` returns false even after edits.
- **Suggested fix:** Wrap each mutation in `history.execute({ do, undo, label })`. v0.1.5 story-composer-01 had this pattern (e.g., addTextOverlay at story-composer-01.tsx:260). Restore equivalent wrapping inside MediaEditor01. Tag v0.1.1.

#### F-02 — `setMode` workaround in story-composer-01 wrapper via `loadState`

- **Severity:** 🔸 Medium
- **Dimension:** Public API (handle completeness)
- **Location:** [src/registry/components/media/story-composer-01/story-composer-01.tsx:200-205](../../../../src/registry/components/media/story-composer-01/story-composer-01.tsx#L200)
- **Observed:** `MediaEditor01Handle` does NOT expose a public `setMode` method (description §7 enumerates 22 methods explicitly without it). The story-composer wrapper's `toComposerCtx.setMode` workaround round-trips through `loadState({...getState(), mode: m})` to set the mode externally. Works at runtime but is internally inconsistent — every other state field has a setter, mode does not.
- **Suggested fix:** Add `setMode(mode: ComposerMode | null) => void` to `MediaEditor01Handle` in v0.3.0 (handle expansion = minor bump). Story-composer wrapper drops the loadState workaround. Update description §7 + plan to enumerate 23 methods.

#### F-03 — `StoryComposer01Labels` cast in MediaEditor01 root is type-unsafe

- **Severity:** 🔸 Medium
- **Dimension:** Public API (label surface)
- **Location:** [src/registry/components/media/media-editor-01/media-editor-01.tsx:201](../../../../src/registry/components/media/media-editor-01/media-editor-01.tsx#L201) (`as Partial<StoryComposer01Labels>` cast)
- **Observed:** MediaEditor01 accepts `labels?: Partial<MediaEditor01Labels>` (nested shape per description §11). Internally it casts to `Partial<StoryComposer01Labels>` (flat) to merge with INTERNAL_LABELS_FALLBACK, because the part files (EditorCamera / EditorToolbar / etc.) still take the flat shape from v0.1.5. The cast loses type-safety — passing nested-shape `labels` props doesn't actually override anything until the parts get refactored. The `StoryComposer01Labels` shim is `@internal` per types.ts:308 and explicitly scheduled for removal.
- **Suggested fix:** Refactor the parts to consume `MediaEditor01Labels` directly. Drop INTERNAL_LABELS_FALLBACK in favor of an immutable DEFAULT_LABELS derived from the nested shape. Tag v0.2.0 or v0.3.0 alongside the setMode handle addition.

#### F-04 — Visual regression vs v0.1.5 story-composer-01 not verified

- **Severity:** 🔹 Low (process gap)
- **Dimension:** Verification
- **Location:** Plan §C18 deliverable
- **Observed:** Plan §C18 calls for walking every v0.1.5 demo tab and confirming identical behavior post-wrapper. The type-export portion of C18 was completed via snapshot probe (73/73 resolve), but the visual portion was deferred to user post-push per session decision. No automated UI regression test exists.
- **Suggested fix:** Document the 5-tab walkthrough script in the C22 decision file; user runs `pnpm dev` post-push and reports any regressions. If a regression surfaces, fix in v0.2.1.

#### F-05 — Plan-vs-implementation drift for video perf-shortcut wording

- **Severity:** 🔹 Low (doc sync)
- **Dimension:** Public API (documentation accuracy)
- **Location:** [plan.md C10 row](../media-editor-01-procomp-plan.md) vs [media-editor-01.tsx exportVideo](../../../../src/registry/components/media/media-editor-01/media-editor-01.tsx) `hasAnyOverlay` check
- **Observed:** Plan §C10 says "Video perf-shortcut: if `enabledTools` lacks overlays, skip composite-video.ts re-encode" — a static gate on the configured capability. Implementation gates on runtime state: `!hasAnyOverlay(state) && !state.crop` — supersets the static gate (fires more often, never less). meta.ts was updated to match the runtime gate at C13 review, but plan.md still reads the static gate. Documentation drift.
- **Suggested fix:** Append a note to plan §C10 indicating the runtime-gate decision. Tag v0.1.1 docs sweep.
