### Rotating dimension

- **Chosen:** **Robustness** (per plan §17.10).
- **Why:** the never-forced skip path, the unresolved-milestone graceful degradation, the deterministic beat derivation, and the fire-once telemetry are the failure surface — a "name required" regression or a crash on bad data is the way this component goes wrong.
- **Verified?** yes (code + built-output inspection; live walkthrough is F-02)
- **Notes:**
  - **Never-forced skip (hard).** Title = `resolveQuestTitle` (`questName?.trim() || name`) — pure, blank/whitespace → team name. `saveDraft` calls `onQuestNameChange(draft.trim())`; a blank draft ⇒ `""` ⇒ the host clears `questName` ⇒ title reverts. **No required-field validation, no nag, no blocking prompt** anywhere in the type system or the code. The default-state invitation is a quiet ghost button, not a modal. Edit affordance gated on `canEditName = editableName && onQuestNameChange != null`. ✅
  - **Unresolved `milestoneId` (graceful).** `deriveBeats` treats an unresolved milestone as not-done → it becomes current/upcoming; flagged `unresolved: true`; `TeamQuestChapters` dev-warns once (NODE_ENV-guarded, in a function body) — **never throws**. Verified with the `CHAPTERS_WITH_UNRESOLVED` fixture. ✅
  - **Deterministic beats.** Stable sort on `NarrativeChapter.order` (`Milestone.order` tie-break); `current` = the first not-done by order; **all-done → no current**; **none-done → first is current**. Order-independent, memoized on `[chapters, milestones]`. ✅
  - **Fire-once telemetry.** ONE IntersectionObserver over all `[data-chapter-id]` nodes (threshold 0.5), a `useRef<Set>` of emitted ids → re-render / scroll-back / re-entry never re-emit; SSR/jsdom guard (`typeof IntersectionObserver`); click fallback (`fireForChapter`) shares the same Set. Re-observes on `chapterKey` change. No `onEvent` → no observer. ✅
  - **SSR-safe:** title is a pure prop function; `doneAt` uses a deterministic UTC formatter (no locale/TZ hydration mismatch — the trophy-shelf pattern); IO in an effect; draft/editing init `false`. No `Date.now()`/random during render. ✅

**Compound structure:** correct light compound — `Root` (all state + `forwardRef` handle) + flat `TeamQuestNameEditor` / `TeamQuestChapters` + 5 Tier-C primitives + logic-free assembly + exported resolvers. Subset paths (timeline-only, name-only) proven in the demo. No `React.lazy` (no heavy dep — stated). The `input` primitive lives only in `QuestNameField`, so the timeline-only subset never pulls it.

## Intentional deviations from the plan (loud, not silent)

1. **`TeamQuestHeader` cut** (plan §5.4 recommended deferring unless the demo showed a real header/body split need). Not built — the assembly + `className` cover it. Keeps the surface minimal (the four mandatory parts + five Tier-C primitives).
2. **Blur does not save** (plan §6.2 allowed "blur-with-change" as a save trigger). Omitted deliberately: saving on blur races the Cancel button (clicking Cancel blurs the input first → would save before cancel fires). Save is `Enter` / the Save button only; `Escape` / the Cancel button cancels. Strictly safer, no behavior lost.
3. **Imperative handle included** (plan §13 recommended shipping it) — `focusNameEditor()` + `scrollToChapter(id)` via `forwardRef` on Root, forwarded by the assembly. Small, real onboarding/deep-link use.
