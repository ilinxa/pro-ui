## Browser walkthrough fixes (2026-06-10) — F-06 closed

The live walkthrough (the de-facto visual gate) found + fixed 7 runtime issues, all of which passed tsc/lint/build clean — reaffirming the "visual walkthrough is the de-facto gate" lesson:

1. **SSR hydration mismatch** — `@dnd-kit` derived `aria-describedby` ("DndDescribedBy-N") from a module-level counter that incremented in different order server vs client. Fix: `useId()`-derived stable `id` on `DndContext` (`media-library-root.tsx`). *(Same class as the rich-card auto-id SSR lesson: any id minted during render must be deterministic.)*
2. **Thrown viewer could crash the page** — added a `PreviewErrorBoundary` around the dispatched viewers (`file-preview.tsx`): a viewer that throws degrades to a graceful "Couldn't load + Download" card.
3. **PDF worker fetch failure + no async error path** — added a `pdfWorkerSrc` prop (threaded root → context → details-pane / lightbox / dispatcher) and wired react-pdf's async `onError` to the fallback (a boundary can't catch the async failure). Demo points at a jsdelivr worker.
4. **Sidebar had no root** — added a synthetic "Library" root node (expanded) so root is visible + clickable.
5. **Sidebar drag did nothing** — `file-tree` (native HTML5 drag) was draggable but its `onMove` was never wired → now `onMove → ctx.moveTo` (drop a folder/file onto a folder, or onto "Library", to move).
6. **Folders shown as "(empty)" when non-empty** — `foldersOnly()` had stripped files, leaving folders with `children:[]`. Sidebar now shows the **full tree**; "(empty)" is honest. Activation guarded (folder → navigate, file → preview).
7. **Cards crushed by the preview pane** — grid used viewport breakpoints (`lg:`/`xl:`) that don't know the sidebar + details pane eat ~510px, so it over-columnized and folder names collapsed. Fix: width-aware `auto-fill minmax()` grids. Also hardened the grid drag (read dragged ids off the drag event, not lagging React state; `pointerWithin` collision; visible `cursor-grab` grips).

All re-verified clean: tsc 0 / lint 0 / meta-deps 54-54 / build / registry:build (31 files). No public-API removals; additions only (`pdfWorkerSrc`).
