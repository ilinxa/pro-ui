## Findings (max 5)

#### F-01 — Clipboard is keyboard + handle only (no context menu)
- **Severity:** 🔹 Low
- **Dimension:** Public API
- **Location:** `todo-tree.tsx`
- **Observed:** todo-tree has no built-in row context menu; Copy/Cut/Paste are Ctrl+C/X/V + imperative handle. This is the approved scope (avoids pulling a new `context-menu` primitive).
- **Suggested fix:** None required; a row context-menu with Copy/Cut/Paste is a future enhancement if demand appears.

#### F-02 — `priorityOptions` only reaches the editor card
- **Severity:** 🔹 Low
- **Dimension:** Public API
- **Location:** `todo-tree-with-editor.tsx`
- **Observed:** The base tree row doesn't render priority; `priorityOptions` is threaded to the `TodoTreeWithEditor` edit card (parity with rich-card/gantt/calendar). A consumer expecting row-level priority display would see none.
- **Suggested fix:** Document that priorityOptions affects the editor; row-level priority badge is a future enhancement.

#### F-03 — Guide delta pending
- **Severity:** 🔹 Low
- **Dimension:** Planning docs
- **Location:** `docs/procomps/todo-tree-procomp/`
- **Observed:** Guide doesn't yet cover clipboard + priorityOptions; meta updated.
- **Suggested fix:** Add a clipboard + priorityOptions section; v0.3.x doc follow-up.
